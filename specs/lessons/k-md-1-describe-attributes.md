## Setting

Caretaker's Bench's standard scene: the lantern-lit workbench. For this lesson, the bench is pre-staged with **a single object** at the center ‚Äî a **feather** ‚Äî chosen because it has multiple obvious, unambiguous attributes (light, soft, smooth) and at least one obvious non-attribute (heavy).

- **Object**: a long, slightly curled brown-and-cream feather laid flat on the bench.
- **Attribute word tags** floating nearby: `light`, `smooth`, `soft`, `heavy`, `rough`. (5 tags total: 3 applicable, 2 non-applicable.)
- **Applied-attributes zone**: a soft glow ring around the feather indicates where dropped tags land. As tags settle, they arrange in a small "tags applied" stack beneath the feather.

Per `k-activity-patterns.md`: standard K HUD, Sanctuary narrator, Buddy at lower-left on an overturned crate.

## Phase 1 ‚Äî I-Show (‚âà25 s)

The kid watches. The pointer drags three applicable tags onto the feather, with one wrong-tag demonstration to show what doesn't fit.

### Animation choreography

| Time | Action |
|---|---|
| 0:00 | Scene fades up. Feather at bench center. 5 word tags float nearby. Music bed in. |
| 0:02 | Narrator opens. |
| 0:05 | Narrator: "Look at this feather. What words describe it?" Brief camera nudge over the feather. |
| 0:07 | Glowing pointer drifts to the tag row, hovers over `light`. |
| 0:08 | Pointer drags `light` ‚Üí drops on the feather. Soft "yes" chime. Tag settles in the applied stack. Narrator: "Feathers are light." |
| 0:11 | Pointer drifts to `heavy` (intentional wrong choice for the demonstration). Drags toward feather. |
| 0:13 | Pointer drops `heavy` on the feather. Soft "nope" puff. Tag returns to its floating position. Narrator: "Heavy? No ‚Äî feathers are not heavy." |
| 0:16 | Pointer drifts to `soft`. Drags ‚Üí drops on feather. Chime. Tag settles. Narrator: "Soft." |
| 0:19 | Pointer drifts to `smooth`. Drags ‚Üí drops on feather. Chime. Tag settles. Narrator: "Smooth." |
| 0:21 | All three applied tags pulse together briefly. Narrator delivers the closing insight. |
| 0:25 | "I've got this" button has been visible since 0:17. I-Show ends. |

### Narration script

> *(0:02)* "Look at this feather. What words describe it?"
>
> *(0:08, after `light` snaps)* "Feathers are light."
>
> *(0:13, after `heavy` rejects)* "Heavy? No ‚Äî feathers are not heavy."
>
> *(0:16, after `soft` snaps)* "Soft."
>
> *(0:19, after `smooth` snaps)* "Smooth."
>
> *(0:21, with all three pulsing)* "Light, soft, smooth. One feather, many words. That's how we describe things."

### Notes for the narrator
- "Light, soft, smooth ‚Äî one feather, many words" is the kid-language version of "an object can have several measurable attributes." Land the "many words" beat slowly ‚Äî this is the standard's load-bearing idea (several attributes for one object).
- The wrong-`heavy` beat is essential: it shows the kid that wrong drops are normal and corrected gently, not punished.
- "That's how we describe things" generalizes beyond the feather.

## Phase 2 ‚Äî We-Try (‚âà25 s)

The kid drags one or two tags onto a fresh object; system completes.

### Setup

| Element | State |
|---|---|
| Scene | Workbench. Feather is gone; a **glow-stone** (a small glowing teal stone) is now centered. Fresh tag set: `shiny`, `smooth`, `dull`, `heavy`, `light`. |
| Glow target | The `shiny` tag halos softly. |
| Pointer | Gone ‚Äî the kid's stylus is the pointer now. |

### Choreography

| Time | Action |
|---|---|
| 0:00 | Narrator: "Your turn. This is a glow-stone. Find words that describe it." `shiny` tag halos. |
| ‚Üí kid drags | Kid drags any tag toward the glow-stone. |
| | If `shiny` (or `smooth`, both applicable) ‚Üí snap with chime; narrator names the attribute: "Shiny." (or "Smooth.") |
| | If `dull` ‚Üí soft "nope," tag returns. Narrator: "Glow-stones aren't dull ‚Äî they shine." |
| | If `heavy` ‚Üí soft "nope," tag returns. Narrator: "Heavy? This little stone is light." |
| +0.5 s | Halo moves to a second applicable tag (whichever the kid hasn't placed yet ‚Äî `smooth` or `light`). Narrator: "Now another." |
| ‚Üí kid drags | Kid drags second tag. Same correction patterns. |
| +0.5 s | System auto-completes by drifting the third applicable tag onto the stone via pointer animation. Narrator names it. |
| 0:22 | All applied tags pulse. Narrator: "Three words for one stone. Well described." |
| End | We-Try ends; advance to You-Do. |

### Hesitation rule
After **5 s** of no drag activity, the halo on the next-suggested tag brightens and the narrator gently repeats the cue. After another 5 s, the system auto-drags the next tag. Lesson advances either way.

### Pass condition
Any kid drag onto the glow-stone (correct or wrong) counts as participation. The lesson advances regardless of perfect accuracy.

### Narration script

> *(start)* "Your turn. This is a glow-stone. Find words that describe it."
>
> *(after correct drop)* "{Shiny / Smooth / Light}."
>
> *(after wrong `dull` drop)* "Glow-stones aren't dull ‚Äî they shine."
>
> *(after wrong `heavy` drop)* "Heavy? This little stone is light."
>
> *(prompting second tag)* "Now another."
>
> *(auto-complete final tag)* "{remaining attribute name}."
>
> *(closing)* "Three words for one stone. Well described."

## Phase 3 ‚Äî You-Do (one round of Caretaker's Bench)

Hand off to the activity's `describe-attributes` sub-mode for a real round.

### Round parameters

- **Sub-mode**: `describe-attributes`
- **Object**: an **apple** (fresh object, not seen in I-Show or We-Try)
- **Available tag pool**: `smooth`, `shiny`, `heavy`, `light`, `rough`, `dull` (6 tags total: 3 applicable to the apple ‚Äî `smooth`, `shiny`, `light`; 3 not ‚Äî `heavy`, `rough`, `dull`)
- **Target**: place at least **2** correct attribute tags (per K.MD.1's "describe *several* attributes" baseline)
- **CPA layer**: Concrete (tags show icon + text per `caretakers-bench.md`)
- **Max attempts**: **3** per `micro-lessons.md` first-encounter forgiveness
- **Hints**: enabled at K standard thresholds

### Pass outcome
- Kid places at least 2 correct tags on the apple within the attempt budget.
- `mastery.standard_practicing` fires for K.MD.1.
- Library entry created.
- `firstEncounter` for K.MD.1 flips to `false`.

### Fail outcome (3 wrong tags placed without 2 correct, or kid taps Done without 2 correct)
- Library entry filed.
- `firstEncounter` flips to `false`.
- Mastery status stays `Introduced`.
- Dashboard flag.

## Edge Cases

- **Kid drops a tag away from the object** ‚Äî tag returns to its floating position. No round-level penalty.
- **Kid drops the same correct tag twice** ‚Äî second drop is a soft no-op (tag is already applied).
- **Object that takes a subjective attribute** ("pretty," "fluffy") ‚Äî not in the lesson tag pool. Per `caretakers-bench.md`, launch tag pool stays measurable/observable only.
- **The "heavy on feather" demo in I-Show reads as "the system makes mistakes"** ‚Äî that's intentional; the demonstration normalizes the soft "nope" + tag-return behavior so the kid isn't surprised when their own wrong drops do the same thing.
- **Glow-stone visual ambiguity** ‚Äî `light` could mean "not heavy" or "emits light." The lesson uses `light` in the "not heavy" sense (per K.MD.1's "weight" attribute). The glow-stone is unambiguously light-in-weight and visually glowing. Confirm in playtest that the dual meaning doesn't confuse kids; if it does, swap to a less-luminous fresh object.
- **Multi-tag application visual stack** ‚Äî applied tags arrange beneath the object as a horizontal row. Confirm the row doesn't get crowded as more tags are applied; consider a small overflow handler if more than 4 tags land on one object (challenge variant only).

## Open Questions

- **Tag count in I-Show** ‚Äî 5 tags (3 applicable, 2 non-applicable) chosen so the kid sees both correct snaps and the wrong-tag demo. Could trim to 4 (3 + 1) to simplify; suggest 5 for the variety.
- **Sequence of correct tags shown** ‚Äî `light`, then `soft`, then `smooth`. Order isn't load-bearing pedagogically; chosen for vocal cadence. Confirm in playtest.
- **Wrong-tag demo placement** ‚Äî current spec: between `light` (correct 1) and `soft` (correct 2). Alternative: at the end, after all 3 correct. Trade-off: showing wrong early normalizes the behavior before any kid does it; showing it last is less interruptive of the "many correct words" insight. Suggest current ordering; revisit if playtest finds the early wrong-demo disrupts.
- **Object roster** ‚Äî feather (I-Show), glow-stone (We-Try), apple (You-Do). All are distinct and have multiple obvious attributes. Confirm the apple has clear correct/incorrect tags in the production tag set; if "round" or "red" need to enter the tag pool eventually for non-K.MD.1 standards, the apple may need its tag list audited.
